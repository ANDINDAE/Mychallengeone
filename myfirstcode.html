What is programming

Programming is the process of designing, writing, testing, debugging / troubleshooting, and maintaining the source code of computer programs. This source code is written in a programming language.

A programming language is an artificial language designed to express computations that can be performed by a machine, particularly a computer. programming languages include;

  • High-level languages, such as BASIC, C, C++, COBOL, FORTRAN, Java, and Pascal.
  • Machine language which consists of numbers only
  • Assembly languages.
The C Programming Language

The C programming language was called "C" because many features derived from an earlier compiler named "B", which in turn was developed from BCPL. BCPL was a simpler version of a language called CPL, which owed many of its features to a language called Algol 60. The names most associated with this early development of C were Ken Thompson and Dennis Ritchie. The main features of C are:

  • Efﬁciency-C has commands that are directly oriented towards the low level instructions, and can often produce very fast and small code.
  • Portability-C is available on virtually every platform.
  • Simplicity-C has a relatively small number of components

C Compilation Model

The Preprocessor accepts source code as input and is responsible for removing comments and interpreting special preprocessor directives denoted by #.

The C compiler translates source to assembly code. The source code is received from the preprocessor and the assembler creates object code.

If a source file references library functions or functions defined in other source files the link editor combines these functions (with main ()) to create an executable file. External Variable references are resolved here also.

C-Basic Data types

C has a concept of 'data types' which are used to define a variable before its use. The definition of a variable will assign storage for the variable and define the type of data that will be held in the location.

C has the following basic built-in or primitive data types.
  1. int
  2. float
  3. double
  4. char
  5. string–uses a sequence of characters
Please note that there is no Boolean data type. C does not have the traditional view about logical comparison, but that’s another story.

Error-ﬁnding (debugging)tools

There are many ways to ﬁnd errors in your programs, but it is much easier to not put them there in the ﬁrst place (and hence the comments given before about designing before building software).

There have been many studies that demonstrate that the cost of ﬁxing an error increases the longer it is allowed to stay in a program. This is particularly true for structural or design errors.

There are a range of tools and mechanisms for ﬁnding errors;

  1. Put in lots of print f statements throughout your program. This allows you to match the expected behavior of your program with its actual behavior. You can also print out the contents of important variables as the program runs.

  2. Use a debugging tool such as gdb which allows you to; access a single step programs,display variable values,modify variable values

Headers vs header ﬁles

Every function in a program has a header part and a body.

The header has the name and type of the function, along with the parameters to that function.

The body is enclosed in braces, and has both declarations and executable statements.

Header ﬁles are declarations that belong to the libraries that come with C. They contain standard C deﬁnes, variable declarations, sometimes code, but more commonly declarations of external functions which are linked into our programs at link time. Some common header ﬁles are:

  • Macros and numbers for error conditions
  • The prototypes for math library functions
  • Standard C types
  • C input and output (such as print f)
  • The prototypes for string functions
    • When we use #include or any other header ﬁle, the ﬁle is included into our program using the preprocessor.

Libraries

Large parts of many programs do the same things over and over again. Commonly used routines are put in libraries, which may be linked in at run time.

There are three distinct advantages of libraries in C programming;

  • Saving on disk space
  • Routines can be updated
  • Routines are more reliable

Most compilers can also statically link in the needed libraries if necessary. The executables are larger. In general you should not bypass library or system calls. If you do, your programs date quickly. When we use a library routine, it saves us having to create a lot of new code.
